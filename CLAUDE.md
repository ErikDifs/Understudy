# Understudy

AI replicas of workers that can stand in for non-critical meetings, then report back.

## What this is

A user delegates a meeting to their AI "Understudy" instead of attending in person. The bot joins the call, follows the user's pre-set instructions, and returns a summary — catching the moments that needed the real person, ignoring the ones that didn't.

The second dimension is proactive networking: the Understudy enters a shared Realm, finds agents whose owners are building toward the same specific problem, and reports back with a Match worth a real conversation.

---

## Tech stack

- **SvelteKit** (Svelte 5) — framework, routing, server actions
- **Tailwind CSS v4** — styling; no component library
- **Drizzle ORM** + **PostgreSQL** — database; `pnpm db:push` to sync schema, `pnpm db:studio` for GUI
- **Recall.ai** — bot SDK for joining meetings and streaming transcripts via webhook
- **Anthropic SDK** (`claude-sonnet-4-6`) — summarization, Intake chat, escalation detection, Profile synthesis
- **Vercel** — deployment target (`@sveltejs/adapter-vercel`)

Local DB: `pnpm db:start` (Docker Compose). Env vars: see `.env.example`.

## Domain model

**Understudy** — the AI replica of a specific user. One per user. Same agent for meetings and networking — different tasks, shared persona and Profile.

**Profile** — the portable representation of the user. Stored as a `synthesized_document`: dense natural language, regenerated by Claude after each Intake session. Injected into every AI context: meeting summarization, escalation checks, realm conversations.

**Intake** — the ongoing in-app chat between the user and their Understudy that builds and refines the Profile. Raw message history is stored as ground truth; the Profile is derived from it. Not a one-time form — it deepens over time.

**Escalation** — a mid-meeting signal that the real user needs to be pulled in. Only fires when the meeting surfaces something not already settled in the Profile. Delivered as an in-app notification via SSE on the meeting detail page.

**Realm** — the closed, AI-only shared space where Understudies network autonomously. No humans, no external protocols.

**Mission** — a networking run. Launched explicitly by the user. Runs persistently until paused or terminated.

**Alignment** — the specific overlap the networking agent hunts for. Not general interest overlap ("both building AI") — same specific problem, complementary constraints, or one agent has already solved what the other is stuck on.

**Match** — a person flagged after finding Alignment. Comes with: who, why, and the key moment from the conversation.

**Report** — what the agent sends back after a Mission. Daily digest or immediate notification on a strong Match.

## Code style

- TypeScript everywhere; no `any` unless interfacing with an untyped third-party payload
- Svelte 5 runes (`$props`, `$state`, `$derived`) — not the legacy Options API
- Server logic lives in `+page.server.ts` and `+server.ts`; keep `.svelte` files presentational
- Drizzle queries in server files directly — no repository abstraction layer unless a query is reused in 3+ places
- No comments unless the WHY is non-obvious; well-named identifiers carry the what
- No unused exports, dead code, or backwards-compatibility shims

---

## UI requirements

- **Mobile-first**: all layouts must work at 375px and up before desktop is considered
- **Accessible**: semantic HTML, keyboard navigable, sufficient color contrast (WCAG AA minimum), `aria-` labels where native semantics fall short
- **Tailwind v4**: use utility classes directly; no custom CSS unless Tailwind cannot express it
- No UI component library — keep it close to native HTML elements with Tailwind styling
